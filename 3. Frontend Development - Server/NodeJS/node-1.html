            Console (screen, keyboard, mouse) is the physical device that allows to interact with the computer.
            Terminal (window) is a text input and output environment; a program that acts as a wrapper and allows to enter commands that the computer processes
            Shell is a program that acts as command-line interpreter; processes commands and outputs the results; interprets and processes the commands entered by the user.
            Command line / CLI (Command line interface) is the interface to enter commends for computer to process.

Commands

            echo $0 : Check shell
            echo : print in the terminal whatever parameter
            pwd : print the name of the current working directory
            ls : list information about the file, sort entries alphabetically
            cd : change the shell working directory

            rmdir : remove directory
            touch : create empty file
            rm : remove (unlink) the files
            cp : copy source to dest, or multiple sources to directory
            mv : rename source to dest, or move sources to directory
            head : view beginning of file or piped data
            tail : view end of file or piped data

Git Commands

            git init : create new local repository
            git add : add one or more files to staging
            git commit : commit changes to repository
            git status : tell current branch
            git remote add origin : detail URL of the remote repository
            git remote -v : list current remote repository
            git push : upload commited changes to remote repo
            gut branch : list all available branches on repo
            git checkout : move from one branch to another
            git pull : pull / download code from remote repository and combine it with local repo
            git diff : view differences between current branch and another
            git merge : merge / combine current branch with another
            git log : list all previous commits

PHP or ASP handles a file request

            1. Sends the task to the computer's file system.
            2. Wait while the file system opens and reads the file.
            3. Returns the content to the client.
            4. Ready to handle the next request.

Node.js handles a file request

            1. Sends the task to the computer's file system.
            2. Ready to handle the next request.
            3. When the file system has opened and read the file, the server returns the content to the client.
